# Stakeholder Analysis

## Introduction
Stakeholders are individuals or groups who are affected by or have an interest in the Smart Workspace Manager system.

---

## Primary Stakeholders

### 1. Students / Users
- Use the system to manage documents, projects, and images
- Benefit from organized workspace and productivity improvement

### 2. Developers
- Responsible for building frontend (React) and backend (FastAPI)
- Maintain system functionality and features

### 3. System Administrators
- Manage deployment, database, and system performance
- Ensure system security and stability

---

## Secondary Stakeholders

### 1. Instructors / Faculty
- Evaluate the project for academic grading
- Assess technical implementation and documentation quality

### 2. Future Employers
- Evaluate project as portfolio evidence
- Assess technical skills and system design ability

---

## Stakeholder Needs

- Easy navigation and usability
- Secure authentication and data protection
- Fast and reliable system performance
- Scalable architecture for future improvements

---

## Conclusion
The system is designed to serve academic, professional, and future industry stakeholders by providing a structured and scalable workspace management platform.
