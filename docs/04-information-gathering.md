# Information Gathering

## Introduction
Information was collected to understand user needs and existing systems for designing the Smart Workspace Manager.

---

## Primary Sources

### 1. User Needs
- Students need better organization for notes, files, and projects
- Difficulty managing multiple tools separately
- Need for centralized workspace system

---

### 2. Existing Systems
Analyzed tools:
- Google Drive (file storage)
- Notion (workspace management)
- Trello (task tracking)

### Limitations Found:
- No unified system combining all features
- Requires multiple platforms
- Limited structured project hierarchy

---

## Secondary Sources

### 1. Online Research
Studied productivity tools and SaaS platforms for design inspiration.

### 2. Technology Research
- React for frontend
- FastAPI for backend
- REST APIs for communication
- JWT for authentication

---

## Key Findings
- Users need a unified workspace system
- Simplicity is more important than complexity
- File + project + note integration is essential
- Scalable design is required for future AI features

---

## Conclusion
The system addresses the need for a centralized, structured, and scalable productivity platform.
