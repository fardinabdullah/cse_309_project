# User Journey

## User Scenario

A university student wants to organize a software engineering project using Smart Workspace Manager.

---

## Step 1: Registration

The user visits the platform and creates a new account using their email and password.

**User Goal:**
Access the platform securely.

---

## Step 2: Login

The user logs into the system using their credentials.

**User Goal:**
Access personal workspaces and resources.

---

## Step 3: Create Workspace

The user creates a workspace named "CSE309 Project".

**User Goal:**
Separate project resources from other activities.

---

## Step 4: Create Project

Inside the workspace, the user creates a project called "Smart Workspace Manager".

**User Goal:**
Organize files and notes related to the project.

---

## Step 5: Upload Documents

The user uploads project documents, requirements, and reports.

**User Goal:**
Store important project resources in one place.

---

## Step 6: Add Notes

The user creates notes containing meeting summaries, ideas, and development plans.

**User Goal:**
Keep project-related information organized.

---

## Step 7: Upload Images

The user uploads diagrams, screenshots, and design mockups.

**User Goal:**
Maintain all project assets within the same platform.

---

## Step 8: Search Resources

The user searches for a document using keywords.

**User Goal:**
Quickly locate required information.

---

## Step 9: Monitor Dashboard

The user views the dashboard to see project summaries and recent activities.

**User Goal:**
Track progress efficiently.

---

## Final Outcome

The user successfully manages projects, documents, notes, and images from a centralized workspace, improving productivity and organization.
